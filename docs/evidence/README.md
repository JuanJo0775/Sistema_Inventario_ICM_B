Evidencias generadas por CI

Coloca aquí los artefactos que el pipeline sube para revisión:

- `coverage.xml` — reporte de cobertura
- `pip_audit.json` — reporte de `pip-audit`
- `junit.xml` — resultados de tests (si se habilita)

Estos archivos deben incluirse en el mismo PR o estar accesibles desde el job de CI.
